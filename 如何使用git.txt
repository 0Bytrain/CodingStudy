1、在keil中清除编译
2、在根目录下新建一个.gitignore文件
3、让AI帮忙生成一个.gitignore文件，然后用记事本打开复制粘贴上去
4、在根目录下右键选择 Git Bash here
5、连续三个指令：git init 使仓库初始化；git add . 添加所有文件 git status 检查是否过滤成功（看看是否还有.o文件.obj之类的；其中红色文件是修改了但还没保存到 Git 的文件，绿色文件是已经准备好要提交的文件。）
6、git commit -m "first commit"直接提交后面的双引号可以随便写东西
7、git remote add origin https://github.com/你的用户名/仓库名.git远程绑定仓库，地址换成自己的
8、git branch -M main和git push -u origin "main"
之后想要更新代码只要 git status然后git add . （某文件位置，如git add Core/Src/main.c）再git commit -m "功能更新"最后git push就可以了。如果更改多个文件就一个个上传写“日志”
注意：如果可以以后不要无脑用git add .因为这样提交写评论时会把所有文件的评论改了，我们只git add修改的文件然后提交评论



进阶：
一般都会先创建一个分支，平常都是在分支上写代码，等到调试完毕发现没问题再在github官网上合并。（创建分支后github上会有一个Compare&pull request）；最后这个分支任务完成了，就用git pull把main中的代码拉下来，然后就可以销毁了（git branch -d <你的旧分支名>）。记得除了删除本地还要删除云端：git push origin --delete <你的旧分支名>。
之后就可以创建其他分支开发其他功能；

git checkout -b name这个指令可以创建分支；然后推送云端：git push -u origin name
分支还有一个很好的功能是可以随时切换；平常在根目录打开git bash后输入git checkout  ... 然后就可以切换至某个分支甚至以前的历史版本（输入哈希值就可以）；

除了用checkout可以看到以前历史代码外，还要会用git restore .这个可以把你的代码恢复成上一次提交的状态（不要用pull），如果你已经提交了，那么就使用（git reset --hard 历史哈希值）
这可以把这个值之后提交的代码直接粉碎，回到当前历史的代码。

若要在一个仓库上传一些毫无关联的小项目只需要在大文件夹里面创建两个独立的小文件夹然后在大文件夹作为根目录就可以，切记.gitignore要把所有项目想要忽略的全部包含进去



报错：
如果创建时勾选了ADD README那么仓库就会多出一个文档，但是本地却没有就会冲突，所以需要先把仓库里的文档拉下来到本地，再重新传输；使用
git pull origin main --rebase

我对git bash使用了：git config --global core.quotepath false
--global：代表全局生效。只要敲这一次，以后电脑上的所有 Git 仓库遇到中文名都不会再变成数字了。
core.quotepath false：意思是关闭路径转义。
执行完这行命令后，再次输入 git status，就能看到正常的中文文件名了！
